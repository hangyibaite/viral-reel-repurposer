# Reference patterns library

Reusable structural shells observed in proven viral content. When you spot one in the original, name it and preserve the shell. Swap only the words inside the placeholders.

Each pattern is presented as:
1. The function (what the pattern does)
2. The shell (the preserved structure)
3. What flexes inside the shell
4. An example of repurposing

---

## 1. Compounding rule-of-X

**Function:** Builds rhythm. Lands hardest on the final item. The intensifier on the last beat is the punchline.

**Shell:**
```
It's how we got [X1]. It's how we got [X2]. It's how we got [X3]. It's how we got fucking [PUNCHLINE].
```

**Flexes:** The items in [X1], [X2], [X3], and [PUNCHLINE]. The intensifier ("fucking", "literal", etc.) stays.

**Anchor:** The intensifier on the last beat is non-negotiable. So is the repeated phrase opener.

---

## 2. Identity reframe

**Function:** Flips the audience's self-perception. Earns the rest of the script by making the viewer the protagonist.

**Shell:**
```
You, who [DENIES being NEW IDENTITY], because I got news for you. You ARE [NEW IDENTITY].
```

**Flexes:** What the audience denies, and what they actually are.

**Anchor:** The "I got news for you" beat. The CAPS on "ARE". The flip happens in one sentence.

---

## 3. Headline drop with verification

**Function:** Thesis statement, delivered as a verdict, followed by a confirmation phrase that demands the viewer agree before continuing.

**Shell:**
```
[THESIS]. [Capisce? / You get it? / Right?] Not only does [THESIS subject + verb], it [AMPLIFIES with comparative].
```

**Flexes:** The thesis, the confirmation phrase choice, and the amplification.

**Anchor:** Short hard sentence + confirmation + comparative amplification. All three beats are present.

---

## 4. Step-by-step with time estimates

**Function:** Tutorial cadence. The time estimate disarms anxiety about complexity. Sometimes paired with a joke to break tension.

**Shell:**
```
Step one, [ACTION]. This takes [TIME ESTIMATE]. [Optional one-liner joke or detail].
Step two, [ACTION]. This takes [TIME ESTIMATE].
Step three, [ACTION]. [DESCRIBE WHAT HAPPENS].
```

**Flexes:** Actions, times, jokes.

**Anchor:** The "Step X, [action]" cadence. Numerical sequencing. Time estimates present (even if approximate).

---

## 5. Soft CTA fumble

**Function:** Anti-guru, low-pressure CTA. Deliberately fumbled phrasing signals "I'm not selling you, just sharing."

**Shell:**
```
I'll put a link in my bio or something to something that where I can like walk you through it or show you how or do a video or something because it's, it's not that deep. It's really not that deep.
```

**Flexes:** Almost nothing. The fumbled phrasing IS the function.

**Anchor:** Preserve the fumble word-for-word. Only swap the destination if the user has a different CTA target.

---

## 6. Reluctant audience hook

**Function:** Pre-empts the dismissal. Creates immediate identity tension. Earns the next 60 seconds.

**Shell:**
```
I know you don't want to hear about [TOPIC], okay? But I need you to lock in and give me [TIMEFRAME] to explain why this is going to change your life, not for the [DISMISSAL] reasons you think.
```

**Flexes:** The topic, the timeframe, the dismissal frame.

**Anchor:** The "I know you don't want to hear" opener. The "not for the reasons you think" closer.

---

## 7. Broad-TOF qualifier list

**Function:** Identifies the audience as universally applicable. Includes a self-deprecating ringer at the end to disarm.

**Shell:**
```
Doesn't matter if you're a [CATEGORY 1], a [CATEGORY 2], you [CATEGORY 3], or even if you're just [SELF-DEPRECATING RINGER].
```

**Flexes:** The categories. The self-deprecating ringer.

**Anchor:** The "doesn't matter if" opener. The ringer at the end is the punchline — don't strip it.

---

## 8. Self-aware meta interruption

**Function:** Speaker calls out that they're doing The Cringe Thing™. Buys patience and disarms the preachy energy.

**Shell:**
```
I am like officially about to [DO THE CRINGE THING], and I swear to god I'm never going to do it again.
```

OR:
```
But I'm about to become the exact kind of person I [USUALLY MOCK], so just let me [REQUEST FORGIVENESS] for [TIMEFRAME].
```

**Flexes:** What the cringe thing is. The forgiveness ask.

**Anchor:** The self-aware energy. The "this is a one-time thing" framing.

---

## 9. Mock-confessional whisper

**Function:** Gossipy / forbidden-knowledge energy. Makes the viewer feel let-in.

**Shell:**
```
[CUTSCENE: lean in, whisper-shout to camera]
"psssst… [TRUTH/SECRET]"
[overlay text: "don't tell [PERSON/GROUP]"]
```

**Anchor:** The whisper delivery. The forbidden-knowledge framing.

---

## 10. Absurd specificity list

**Function:** Escalating absurdity. The list itself is the joke. Each item is more random than the last.

**Shell:**
```
Sorry babe, it's not [THING]. It's [THING], [RANDOM RELATED THING], [EVEN MORE RANDOM THING], and probably [ABSURD ADDITION] too.
```

**Anchor:** The escalation. The "and probably [most random] too" closer.

## 11. Shuddup energy

**Function:** Pre-empts the viewer's objection in the viewer's own voice. Dismisses it immediately.

**Shell:**
```
'[mOcKiNg fOnT QUOTING THE VIEWER'S EXCUSE]—' shuddup. [BLUNT TRUTH].
```

**Anchor:** The mOcKiNg fOnT delivery. The "shuddup" interruption. The blunt-truth follow-up.

---

## 12. Pro tip deadpan

**Function:** Mocks conventional wisdom by stating it back literally. Sets up a real point right after.

**Shell:**
```
Pro tip: [OBVIOUS THING NO ONE NEEDS TO BE TOLD].
[beat]
[REAL POINT or PIVOT]
```

**Anchor:** The "Pro tip" opener with the obvious thing.

---

## 13. Vine boom emphasis

**Function:** Hard punchline after a setup. The sound effect does half the work.

**Shell:**
```
[ONE-WORD OR THREE-WORD VERDICT].
[vine boom sound]
```

**Anchor:** Short verdict. Sound cue. Dead-face delivery.

---

## 14. WAIT WAIT pattern interrupt

**Function:** Pre-empts a misread. Resets pace when the previous beat sounded bad on its own.

**Shell:**
```
[STATEMENT THAT SOUNDS BAD ALONE]
WAIT, WAIT. [CLARIFICATION].
```

**Anchor:** The hard interrupt. The clarification immediately after.

## 15. Mocking impression of the hater

**Function:** Calls out a real reaction the audience would have. Voices it in mocking tone. Dismisses it deadpan.

**Shell:**
```
[mOcKiNg fOnT: "[THE THING THE HATER WOULD SAY]"]
(deadpan return to normal) "Yeah. No."
```

**Anchor:** The mOcKiNg fOnT impression. The deadpan "yeah. no." dismissal.

## How to use this library

1. When mapping the original transcript in Step 4 of the main workflow, identify which of these patterns appear.
2. Mark each pattern's timestamp and content.
3. When repurposing, swap content WITHIN the shell, never rewrite the shell.
4. If you're tempted to rewrite a pattern's shell, stop. The shell is the proof of work. Preserve it.

## When the original uses a pattern not in this library

Some originals will use structures not catalogued here. When that happens:
1. Name what the pattern is doing (its function)
2. Identify the parts that are reusable shell vs. content-specific
3. Preserve the shell, swap the content
4. Note it for future reference (the user may want to add it to this library)
