# Structural plan template

Use this format when outputting the structural plan in Step 5 of the main workflow.

---

```
Structural notes before writing:

- PRESERVING: [name the iconic beats by what they are — hook, closer, rule-of-X structure, etc.]
- SWAPPING: [name the topic-specific things being swapped — tool names, examples, identity reframe target, CTA]
- PARAPHRASE OPTIONS NEEDED: [name the jokes/specifics that need 3-5 options before locking in]
- OPEN QUESTIONS: [anything still ambiguous, or "none"]
- RUNTIME TARGET: [matching original ±20s, or user-specified target]
```

---

## Example (short-form)

```
Structural notes before writing:

- PRESERVING: reluctant-audience hook ("I know you don't want to hear about AI"), self-aware meta beat ("officially about to cross the line"), headline-drop cadence ("X. Capisce?"), iconic closer ("It's not that deep")
- SWAPPING: software → automation framing, tool list (Codex/Replit/Higgsfield/Canva/Claude Code/Claude anything), identity reframe target (software engineer → marketer), CTA (link in bio → comment "brand")
- PARAPHRASE OPTIONS NEEDED: the "what should I make with this food in my fridge" joke
- OPEN QUESTIONS: none
- RUNTIME TARGET: ~1:30 (down from 1:50 by cutting software prologue)
```

---

## Example (long-form)

```
Structural notes before writing:

- PRESERVING: opening claim cadence, all act break transitions, recap moments ("Now, why do you need to care about this?"), thesis line cadence + position at 60% mark, identity reframe shell
- SWAPPING: the thesis content itself (software → marketing automation), all examples in act 2 (Uber/DoorDash → equivalents for new topic), tool list, CTA
- PARAPHRASE OPTIONS NEEDED: the absurd-specificity ChatGPT examples, the self-roast meta beat
- OPEN QUESTIONS: whether to preserve the entire "writing with code" prologue or compress it for the new topic
- RUNTIME TARGET: 2:00 max (down from 2:30)
```
